<bare-select>

The lean alternative to <select>.

  • Easy to customize
  • Keyboard-friendly
  • Simple and lightweight

Styles are your beer

We ship no stylesheets with <bare-select>.

Seriously. It’s up to you how it looks on your website. Like the look above? Grab the stylesheet. Or make it as awesome as you want. It’s easy as pie.

The perfect keyboard UX

We pay special attention to keyboard navigation UX. We’ve modelled it after the native behavior in Chromium.

Easier than ever

  1. Download and import: 
    
<script src="bare-select.drop-in.js"><script>
  2. You’re set!

<bare-select> is a custom element (aka Web Component). Its usage is as easy as <bare-select></bare-select>. No need to write JavaScript to configure, activate and deactivate elements – everything is handled for you dynamically.

By the way, <bare-select> doesn’t ever modify the DOM tree. The drop-in build exports no global variables. As a bonus the thing is hackable down to the bone.

This makes <bare-select> easy to use in any workflow. Whether you’re writing HTML by hand, generating it with JavaScript – or using React, virtual-dom, Angular or even another language like elm.

Falls back to CSS

Here’s how the thing operates before any JavaScript kicks in:

Most existing solutions are heavily based on JavaScript. But <bare-select> falls back gracefully to pure CSS. Why, you ask?

  • It works before all scripts are fully loaded.
  • It plays well with your scripts.
  • It stays accessible to people and machines who don’t use JavaScript.
  • It will still work if a user saves your website locally.
